# ts-server-fn

Wrap dioxus-fullstack server functions and emit a type-safe **TypeScript client**
at macro-expansion time — no separate codegen step.

## What it does

Annotate a server function with `#[get]/#[post]/#[put]/#[patch]/#[delete]`
(same syntax as dioxus-fullstack / by-macros):

```rust
use ts_server_fn::post;

#[post("/api/rooms/{room_id}/comments?after", user: User)]
pub async fn add_comment(
    room_id: RoomPartition,        // path
    after: Option<String>,         // query (skipped when None)
    user: User,                    // server extractor (stripped from client)
    req: AddCommentRequest,        // body → { "req": <value> }
) -> Result<CommentResponse, ApiError> { /* ... */ }
```

The macro does two things:

1. **Re-emits the original server fn unchanged.** It re-attaches
   `#[::dioxus::fullstack::post(...)]` so dioxus generates the server handler +
   browser RPC stub exactly as before. The server keeps working 100% — this
   crate is a transparent wrapper.

2. **Emits a TypeScript client function when generation is enabled** (see below):

   ```typescript
   // AUTO-GENERATED by ts-server-fn. DO NOT EDIT.
   import type { AddCommentRequest } from "../types/AddCommentRequest";
   import type { CommentResponse } from "../types/CommentResponse";
   import type { RoomPartition } from "../types/RoomPartition";
   import { apiPost } from "../runtime/client";

   export async function addComment(roomId: RoomPartition, after: string | null, req: AddCommentRequest): Promise<CommentResponse> {
     const __q = new URLSearchParams();
     if (after !== null && after !== undefined) __q.set("after", String(after));
     const __qs = __q.toString();
     const __url = `/api/rooms/${encodeURIComponent(String(roomId))}/comments` + (__qs ? `?${__qs}` : "");
     return apiPost<CommentResponse>(__url, { "req": req });
   }
   ```

## Triggering generation

Generation is gated on the **`TS_SERVER_FN_PACKAGE_DIR`** environment variable,
read via `std::env::var` **at the consumer's build time** (not `option_env!`),
so toggling it never requires rebuilding this crate:

```bash
# Generates one .ts per handler under $DIR/src/handlers/
TS_SERVER_FN_PACKAGE_DIR=/path/to/ts-package cargo check --features server

# Unset → nothing is written; the server fn is emitted normally.
cargo check --features server
```

> Incremental compilation note: cached handlers are not re-expanded, so a `.ts`
> is only (re)written when its handler is actually recompiled. Treat generation
> as an intentional clean pass (a future `make gen-ts` clears
> `$DIR/src/handlers` first), not something every dev build refreshes.

## Generated vs hand-written

- **Generated** (this crate): `$DIR/src/handlers/<fnName>.ts` — one thin typed
  wrapper per handler.
- **Hand-written** (copy `runtime/client.ts` into `$DIR/src/runtime/client.ts`):
  the fetch transport — base URL, `credentials: "include"`, body wrapping,
  `status → throw ApiError`, optional header injection for mobile.
- **`$DIR/src/types/*.ts`**: produced by `ts-rs` from the Rust DTOs (out of
  scope for this crate; Phase 2 of the asset integration).

## Wire contract

Matches dioxus-fullstack (verified against asset `common/fullstack/server_fn.rs`):

| Aspect | Contract |
|---|---|
| URL | path template `{}`→urlencoded path arg, then `?k=v` query (None skipped) |
| Method | per the attribute |
| Body (POST/PUT/PATCH) | `{ "<argName>": <value> }` — keyed by the body arg's **original snake_case name** |
| Success (2xx) | plain JSON of the return value |
| Failure (non-2xx) | `throw ApiError { status, body }` |
| Auth | same-origin cookies via `credentials: "include"` |

> Exactly **one** body arg is supported (same policy as by-macros).

## Type mapping

| Rust | TS |
|---|---|
| `Option<T>` | `T \| null` |
| `Vec<T>`, `[T]`, `HashSet<T>` | `T[]` |
| `String`, `&str`, `char` | `string` |
| `i8..u128`, `f32`, `f64`, `usize` | `number` |
| `bool` | `boolean` |
| `()` | `void` |
| `HashMap<K,V>` / `BTreeMap` / `IndexMap` | `Record<K, V>` |
| `Form<T>` | `T` (unwrapped) |
| `Result<T, _>` | `T` (error half stripped) |
| named type `Foo` | `Foo` (imported from `../types/Foo`) |
